# 系統架設與運作成本分析報告

本報告旨在評估「12-18 個月市場熱點預見與自動化資訊收集系統」的初始架設成本與長期運作（維護）費用，並提供成本優化策略。

---

## 1. 初始架設與建置成本 (One-time Setup Cost)

系統採用輕量化、無伺服器 (Serverless) 的架構設計，幾乎**零硬體建置成本**：

| 項目 | 描述 | 預估費用 (USD) | 備註 |
|---|---|---|---|
| **開發與測試 API 額度** | 主要用於測試 LangGraph 狀態機的 Agent 提示詞、 Critic 自我修正循環。 | **$5.00 - $10.00** | 視測試次數而定，使用 gpt-4o-mini 可降低此部分 90% 費用。 |
| **程式碼託管與 CI/CD** | 使用 GitHub Repository 與 GitHub Actions 進行每週自動化排程。 | **$0.00** | 適用於 GitHub Free 帳戶（私有/公有倉庫皆提供每月免費 2,000 分鐘 Actions 額度）。 |
| **高頻資料接入** | 接入 yfinance 及模擬/爬取高頻報價數據。 | **$0.00** | 全數使用公開 API 或網頁爬蟲。 |
| **初始建置小計** | | **<$10.00** | 幾乎無初始資金門檻。 |

## 2. 運作與維護成本 (Operational Cost)

運作成本主要來自 **LLM API（如 OpenAI gpt-4o）** 的調用。
假設每週自動執行 1 次，針對 1 個技術板塊（例如「CPO 光電傳輸」）進行完整研判與報告撰寫。

### 2.1 LLM 費用估算（以 OpenAI 官方定價為例）
- **GPT-4o 定價**：
  - 輸入 Token (Input)：$5.00 / 1,000,000 Tokens ($0.005 / K)
  - 輸出 Token (Output)：$15.00 / 1,000,000 Tokens ($0.015 / K)
- **GPT-4o-mini 定價** (備選/優化方案)：
  - 輸入 Token (Input)：$0.150 / 1,000,000 Tokens ($0.00015 / K)
  - 輸出 Token (Output)：$0.600 / 1,000,000 Tokens ($0.0006 / K)

### 2.2 單次執行 (Single Run) Token 消耗拆解
假設系統平均經歷 **1.5 次自我修正循環** (即第一次 Critic 不通過並提供回饋，第二次修正後通過)：

#### 節點 1: 三大專家節點 (Supply Chain, Pricing, Media)
* **輸入**：提示詞 + 收集到的結構化數據 (約 3,000 tokens)
* **輸出**：三位專家的分析意見 (約 1,500 tokens)
* *小計*：3,000 $\times$ $0.005 + 1,500 $\times$ $0.015 = **$0.0375**

#### 節點 2: 報告撰寫者 (Writer)
* **輸入**：專家意見 + 寫作範本與規則 (約 5,000 tokens)
* **輸出**：學術級可行性報告 (約 3,000 tokens)
* *小計*：5,000 $\times$ $0.005 + 3,000 $\times$ $0.015 = **$0.0700**

#### 節點 3: 品質評審 (Critic)
* **輸入**：報告內容 + 審查規格 (約 4,000 tokens)
* **輸出**：Pydantic 結構化審查結果 (約 200 tokens)
* *小計*：4,000 $\times$ $0.005 + 200 $\times$ $0.015 = **$0.0230**

#### 自我修正額外開銷 (以 0.5 次計)
* 修正時將帶入 Critic Feedback 重新生成專家意見與報告。
* *小計*：($0.0375 + $0.0700 + $0.0230) $\times$ 0.5 = **$0.0652**

#### 單次執行總費用 (GPT-4o)
$$\text{單次總花費} = 0.0375 + 0.0700 + 0.0230 + 0.0652 = \mathbf{\$0.1957} \approx \mathbf{TWD\ \$6.40}$$

## 3. 成本優化策略 (Cost Optimization Strategies)

為維持高分析品質且將運作成本降至最低，系統設計將實施以下優化手段：

1. **混合模型路由 (Hybrid Routing)**：
   - **學術寫作 (Writer)**：保留使用 **GPT-4o**，確保繁體中文措辭嚴謹、結構專業。
   - **初期分析與 Critic 審查**：使用 **GPT-4o-mini**。由於 Critic 與專家分析屬於邏輯判斷與格式查核，mini 已具備足夠推理能力，可降低約 75% 的非寫作 token 費用。

2. **啟用 Prompt 緩存 (Prompt Caching)**：
   - 由於每週運行的系統提示詞 (System Prompt) 與寫作規範 (Style Guides) 是靜態且重複的，OpenAI 會自動對大於 1,024 tokens 的重複輸入進行緩存，可享受 **50% 的輸入 Token 折扣**。

3. **本地模型整合 (Local Model Collaboration)**：
   - 根據本機規範，在開發與日常偵錯階段，優先調用本地執行的 **Gemma4** 模型進行 Critic 與代碼邏輯驗證，完全免去 API 費用。

4. **GitHub Actions 運行優化**：
   - 由於 Windows Runner 的計費點數是 Linux Runner 的 2 倍，我們的 CI/CD 排程應設定在 **Ubuntu-latest** 容器環境中運行（相容 Python 及 Bash 檢查），使自動化排程費用完全為 **$0**。
